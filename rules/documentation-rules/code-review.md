# Code Review Best Practices

## Review Guidelines
1. **Be Constructive** - Focus on improvement
2. **Explain Why** - Not just what's wrong
3. **Be Specific** - Point to exact code
4. **Suggest Solutions** - Offer alternatives

## What to Review
1. **Correctness** - Logic, edge cases, bugs
2. **Design** - Architecture, patterns, structure
3. **Readability** - Clear, maintainable code
4. **Tests** - Coverage, quality, effectiveness
5. **Security** - Vulnerabilities, risks

## Review Types
1. **Quick Review** - High-level overview
2. **Detailed Review** - Line-by-line analysis
3. **Security Review** - Focus on vulnerabilities
4. **Performance Review** - Optimization opportunities

## Common Issues
1. **Naming** - Poor or inconsistent names
2. **Comments** - Outdated or unnecessary
3. **Complexity** - Too complex for purpose
4. **Duplication** - Repeated code
5. **Formatting** - Inconsistent style

## Response Time
1. **Initial Review** - Within 24 hours
2. **Follow-up** - Prompt responses
3. **Urgent** - Same day for critical issues
4. **Blocking** - Don't block without feedback
