# Data Privacy Rules

## Data Collection
1. **Minimize Collection** - Only collect what's needed
2. **Explicit Consent** - Users agree to collection
3. **Purpose Limitation** - Use only as stated
4. **Data Minimization** - Anonymize when possible

## Data Storage
1. **Encryption at Rest** - Protect stored data
2. **Access Controls** - Limit who can access
3. **Retention Policies** - Delete when no longer needed
4. **Data Localization** - Store where required

## Data Processing
1. **Purpose Binding** - Use only as intended
2. **Accuracy** - Keep data accurate
3. **Limited Access** - Need-to-know basis
4. **Audit Logging** - Track all access

## User Rights
1. **Access** - Users can see their data
2. **Rectification** - Users can correct data
3. **Erasure** - Users can delete data
4. **Portability** - Users can export data
5. **Objection** - Users can object to processing

## Compliance
1. **GDPR** - European privacy law
2. **CCPA** - California consumer privacy
3. **HIPAA** - Health data protection
4. **SOC 2** - Security controls
5. **PCI DSS** - Payment card data
