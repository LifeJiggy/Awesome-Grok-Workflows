# Bias Mitigation Rules

## Awareness
1. **Acknowledge Bias** - All systems have biases
2. **Diverse Teams** - Build diverse teams
3. **Multiple Perspectives** - Consider all users
4. **Continuous Learning** - Stay informed

## Data Collection
1. **Representative Data** - Diverse, representative samples
2. **Balanced Datasets** - Avoid over-representation
3. **Data Audits** - Regularly audit for bias
4. **Synthetic Data** - When real data is biased

## Model Development
1. **Fairness Metrics** - Measure across groups
2. **Bias Testing** - Test for known biases
3. **Explainability** - Understand why predictions
4. **Human Review** - Human oversight for high-stakes

## Deployment
1. **A/B Testing** - Test with real users
2. **Rollback Plan** - Can quickly remove biased model
3. **Monitoring** - Track performance by group
4. **Feedback Loop** - Accept user feedback

## Documentation
1. **Model Cards** - Document capabilities and limitations
2. **Known Biases** - Document known biases
3. **Use Cases** - Document intended and prohibited uses
4. **Audits** - Regular bias audits
