# JavaScript/TypeScript Style Rules

## General Rules
1. **Semicolons** - Always use
2. **Quotes** - Single quotes preferred
3. **Indentation** - 2 spaces
4. **Line Length** - 100 characters

## Naming Conventions
1. **Variables** - camelCase
2. **Constants** - UPPER_SNAKE_CASE or camelCase
3. **Classes** - PascalCase
4. **Functions** - camelCase
5. **Private** - _leading_underscore

## TypeScript Specific
1. **Type Safety** - No 'any' unless necessary
2. **Interfaces** - Prefer interfaces over types for objects
3. **Enums** - Use const enums
4. **Generics** - Use for reusable components
5. **Strict Mode** - Enable strict type checking

## React Specific
1. **Components** - PascalCase
2. **Props** - camelCase
3. **Hooks** - prefix with 'use'
4. **Files** - Component.tsx
5. **Styling** - CSS-in-JS or CSS modules

## Best Practices
1. **Const** - Prefer const over let
2. **Arrow Functions** - For callbacks
3. **Destructuring** - For objects and arrays
4. **Spread Operator** - For object cloning
5. **Async/Await** - Over Promises
