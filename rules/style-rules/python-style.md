# Python Style Rules

## PEP 8 Compliance
1. **Indentation** - 4 spaces per level
2. **Line Length** - 79 characters maximum
3. **Blank Lines** - 2 blank lines between classes
4. **Imports** - Separate on different lines

## Naming Conventions
1. **Variables** - snake_case
2. **Constants** - UPPER_SNAKE_CASE
3. **Classes** - PascalCase
4. **Functions** - snake_case
5. **Private** - _leading_underscore

## Docstring Style
1. **Module Docstring** - Top of file
2. **Function Docstring** - Describe function
3. **Class Docstring** - Describe class
4. **Google Style** - Popular format
5. **Type Hints** - Include when possible

## Best Practices
1. **List Comprehensions** - Preferred for simple cases
2. **Generators** - For memory efficiency
3. **Context Managers** - Resource management
4. **Type Hints** - For better IDE support
5. **Virtual Environments** - Isolated dependencies

## Code Organization
1. **Imports** - Standard library, third-party, local
2. **Constants** - After imports
3. **Classes** - Next
4. **Functions** - At bottom
5. **Main Guard** - For executable scripts
