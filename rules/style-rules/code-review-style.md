# Code Review Style Rules

## Review Focus
1. **Correctness** - Does it work?
2. **Design** - Is it well-designed?
3. **Readability** - Easy to understand?
4. **Tests** - Are there tests?
5. **Performance** - Is it efficient?

## Comment Types
1. **Nitpicks** - Minor style issues
2. **Suggestions** - Better approaches
3. **Comments** - Why this approach
4. **Questions** - Seek clarification
5. **Blocking** - Must fix before merge

## Constructive Feedback
1. **Be Respectful** - Professional tone
2. **Be Specific** - Point to specific code
3. **Explain Why** - Context for feedback
4. **Offer Solutions** - Suggest alternatives
5. **Acknowledge Good** - Highlight strengths

## Review Process
1. **Timely** - Review within 24 hours
2. **Thorough** - Don't rush
3. **Focused** - Stay on task
4. **Responsive** - Answer questions

## Common Issues
1. **Too Many Comments** - Overwhelming
2. **Vague Comments** - Not actionable
3. **Personal Preferences** - Not blockers
4. **Out of Scope** - Don't expand scope
5. **Holding Hostage** - Don't block for nitpicks
