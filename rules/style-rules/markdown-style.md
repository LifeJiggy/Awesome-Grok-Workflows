# Markdown Style Rules

## Formatting
1. **Headings** - Use # for H1, ## for H2, ### for H3
2. **Line Length** - Wrap at 80 characters
3. **Blank Lines** - Between paragraphs and sections
4. **Code Blocks** - Use triple backticks with language

## Lists
1. **Bullet Points** - Use - or *
2. **Numbered Lists** - Use 1., 2., 3.
3. **Indent** - 2 spaces for nested lists
4. **Consistency** - Same marker throughout

## Code Examples
1. **Language Tag** - Specify language
2. **Comments** - Explain complex code
3. **File Names** - Show file names
4. **Output** - Show expected output

## Links and Images
1. **Descriptive Text** - Clear link text
2. **Relative Links** - Within repository
3. **Alt Text** - For images
4. **References** - For repeated links

## Tables
1. **Alignment** - Left-align by default
2. **Separators** - Use --- between header and data
3. **Width** - Consistent column widths
4. **Brevity** - Don't overcomplicate

## Best Practices
1. **File Extension** - .md or .markdown
2. **Encoding** - UTF-8
3. **No HTML** - Pure Markdown preferred
4. **Check Links** - Verify all links work
